# Week 4 Assessment - Decision Tree Analysis

## Problem Statement
This assessment focuses on using a Decision Tree model in Orange Data Mining to perform root-cause analysis and identify the key factors that influence employee attrition.

## Tool Used
- Orange Data Mining

## Model Used
- Decision Tree

## Target Variable
- Attrition / Employee status: Left or Stayed

## Purpose of Decision Tree
The Decision Tree model helps explain the possible reasons behind employee attrition by splitting the data based on important features. It is useful because the output can be visually interpreted like a set of business rules.

## Interpretation
The Decision Tree helps identify which factors are most important in separating employees who left from employees who stayed.

The top/root node of the tree is especially important because it represents the strongest first-level factor used by the model to divide the data. Further branches show additional conditions that help explain employee movement.

## Business Insight
From a business perspective, the Decision Tree can support HR teams in understanding the possible root causes of attrition. Instead of only predicting whether an employee may leave, the tree gives a clearer explanation of the conditions or factors linked with employee leaving behaviour.

This can help management focus on areas such as employee satisfaction, compensation, workload, career growth, manager relationship, or other key drivers identified in the tree.

- How to select the correct target variable
- How to use Tree Viewer to understand model splits
- How to interpret the root node and branches
- How Decision Tree can be used for root-cause analysis
- How to convert model output into business insights
